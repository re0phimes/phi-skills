# Phi Frontend

## Workflow Path

Default path:

1. `phi-brainstorming`
2. `phi-planning`
3. `phi-execute` or `phi-tdd`
4. `phi-verify`

## Primary Sources

- Workflow owner: `superpowers`
- Domain owner during implementation guidance: ECC `frontend-patterns`

## Secondary Sources

- ECC `coding-standards`
- ECC `e2e-testing`
- `phi-local` project-specific UI conventions

## Trigger Areas

- React or Next.js work
- layout changes
- styling, dark mode, responsiveness
- component boundaries and state flow
- browser interaction bugs

## Handoff

Always finish through `phi-verify` so the request is checked against the original UI and behavior goals.
